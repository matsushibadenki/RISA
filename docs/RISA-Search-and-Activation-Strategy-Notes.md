# RISA 探索・活性化戦略メモ

## 1. 目的

この文書は、RISA の弱点として予想される

- 事前学習の遅さ
- 推論速度の低下
- 探索爆発
- 構造爆発

に対して、どのような改善原理が考えられるかを整理するための設計メモです。

現時点の主張は一つです。

> RISA のボトルネックは、構造更新そのものよりも、どこを探索するかを決める原理が未確立であること

です。

ANN において Backpropagation や Attention が実用化の鍵だったように、RISA においても **構造探索・局所活性化・構造編集の制御原理** が最大のブレークスルー候補になります。

---

## 2. 全探索をしない

最重要原則はこれです。

RISA は巨大グラフ全体を毎回見てはいけません。

例えば 100 億ノードがあっても、入力が `犬` なら最初から

- `動物`
- `哺乳類`
- `ペット`
- `走る`

周辺だけを起動するべきです。

これは単なる graph search ではなく、**local activation** として考える方が自然です。

### 設計含意

- 入力から即座に候補近傍を引く仕組みが必要
- 起動サブグラフのサイズに上限を持つべき
- 編集候補生成も活性化領域内に限定すべき

---

## 3. 局所活性化

RISA の推論は「全体計算」ではなく「局所起動」でなければなりません。

### 基本イメージ

```text
入力: 犬
  ->
犬
  ->
動物
  ->
哺乳類
  ->
行動
```

この局所活性化が成立すれば、探索量は

```text
100億
  ->
100
```

のように大幅に減る可能性があります。

### 今後の実装論点

- 活性化の開始点をどう決めるか
- 活性の拡散半径をどう制御するか
- 文脈によって活性経路をどう切り替えるか
- どの時点で探索を打ち切るか

---

## 4. インデックス構造

DB が全件検索を避けるのと同様に、RISA も入口インデックスを持つべきです。

例:

- `動物`
- `色`
- `形`
- `行動`
- `感情`

などのカテゴリ軸を入口として持ち、入力に応じて探索開始領域を限定します。

### 期待される効果

- 初期候補数の削減
- 不要な比較の回避
- 局所活性化の起点安定化

### 設計含意

- ノードラベルだけでなく、型・文脈・機能役割によるインデックスが必要
- 将来的には複数インデックスを併用すべき

---

## 5. 階層探索

概念が階層構造を持つなら、探索も階層的にできます。

```text
生命
  ->
動物
  ->
哺乳類
  ->
犬
```

この場合、探索は関連階層だけを辿ればよく、全候補比較を避けられます。

これは B-tree 的な発想に近いです。

### 設計含意

- 概念層の明示化が必要
- 上位概念と下位概念の接続を安定に保つ必要がある
- 階層が深くなりすぎる場合の圧縮方針が必要

---

## 6. ショートカットとキャッシュ

頻繁に使う経路は毎回探索せず、ショートカットとして保持すべきです。

例:

```text
犬
  ->
動物
```

を何度も使うなら、

```text
犬
  =>
動物
```

のような高速経路を持ってよい。

### 期待される効果

- 推論速度向上
- 頻出経路の安定化
- 活性化時の候補絞り込み

### 注意点

- キャッシュが古くなった場合の更新規則が必要
- 例外文脈を潰さないようにする必要がある

---

## 7. 圧縮の徹底

探索量を抑えるには、そもそも構造数を増やしすぎないことが重要です。

例えば

```text
犬
猫
馬
牛
羊
ヤギ
```

を個別に強く扱い続けるより、

```text
哺乳類
```

のような再利用構造へまとめた方が探索も記憶も軽くなります。

### 設計含意

- 抽象化は意味理解だけでなく探索効率化の役割を持つ
- 圧縮失敗は単なる記憶肥大ではなく推論速度低下にも直結する

---

## 8. 休眠と忘却

使われない構造は即削除ではなく、まず休眠化する方が良い可能性があります。

```text
長期間未使用
  ->
休眠
  ->
必要時のみ復帰
```

これはストレージ階層に近い発想です。

### 期待される効果

- 活性探索対象の縮小
- 重要構造の保持
- 不要構造の常時コスト削減

### 実装含意

- 活性層と休眠層の分離が必要
- 復帰条件を明確にする必要がある

---

## 9. 比較対象を減らす

RISA はノード全体を比較するのではなく、**変化や役割だけを比較する** 方が良い可能性があります。

例えば、

```text
犬
猫
馬
```

を直接比べるより、

```text
食べる
  ->
満腹
```

のような変化パターンだけを見る方が比較量を大幅に減らせます。

### 含意

- 概念形成は属性一致より状態遷移一致を優先する
- 比較単位をノードからイベント構造へ移す

---

## 10. イベント中心化

RISA はノード中心よりイベント中心の方が探索効率に向くかもしれません。

例えば、

```text
犬
```

よりも

```text
犬が走る
```

を主キーにする方が、起動領域を自然に限定できます。

### 含意

- 活性化キーは entity 単体ではなく event signature でもよい
- 学習・予測・説明の共通単位を event に揃えやすい

---

## 11. Relation Attention

Transformer にとって Attention が鍵だったように、RISA にも「どの関係だけを起動するか」を決める原理が必要です。

ここでは仮に **Relation Attention** と呼びます。

### 基本発想

入力 `犬` が来たときに、

- `哺乳類`
- `動物`
- `走る`
- `疲労`

など関連の強い関係だけを優先的に起動する。

### 役割

- 探索空間削減
- 推論速度向上
- 編集候補の局所化
- ノイズ経路の抑制

### 今後の研究課題

- 何をもって「関連が強い」と判断するか
- 文脈で attention をどう変えるか
- 概念とイベントのどちらを優先起動するか

この原理は、RISA の実用性を左右する最重要候補の一つです。

補足:

Relation Attention をさらに進めると、活性化対象はノードではなく「関係の場」そのものとみなせる可能性があります。この見方は [RISA 関係場とイベントパケットのメモ](RISA-Relation-Field-and-Event-Packets.md) に整理します。

---

## 12. 波としての探索

探索を逐次比較ではなく、**波の伝播** として見るアイデアがあります。

例:

```text
犬
  ~~~~~
哺乳類
  ~~~~~
動物
  ~~~~~
生命
```

入力から波が広がり、複数の波が重なった場所を推論候補とみなす。

これは spreading activation に近い発想ですが、RISA では「重み付きニューラル発火」ではなく「関係活性化」として捉えます。

### 利点

- 局所起動と相性が良い
- 関連経路の自然な競合が起こる
- 推論候補を交差点として取り出せる

### 課題

- 波の減衰則
- 文脈による伝播制御
- 拡散しすぎた場合の停止条件

---

## 13. 発達段階

最初からすべてを抽象化しようとすると、RISA は重くなりやすいです。

そのため、発達段階を導入する案があります。

例:

```text
経験100件
  ->
イベントだけ保存

経験1000件
  ->
局所概念を作る

経験10000件
  ->
抽象概念を作る

経験100000件
  ->
理論を作る
```

### 含意

- 抽象化タイミングを経験量で制御できる
- 初期段階の過剰編集を防げる
- 計算資源を学習段階ごとに配分できる

---

## 14. 多層アーキテクチャ

探索と更新を両立するには、単層グラフより多層構造の方が自然かもしれません。

### 14.1 二層案

```text
Layer 1
高速イベント層
数秒〜数日

Layer 2
低速構造層
数日〜数年
```

高速層は頻繁に書き換え、低速層は滅多に変えない。

### 14.2 四層案

```text
Layer 0
Reaction Layer
数ms〜秒

Layer 1
Episode Layer
数分〜数日

Layer 2
Concept Layer
数週間〜数年

Layer 3
Theory Layer
法則・因果・世界モデル
```

### 期待される効果

- 推論は高速層中心で回せる
- 抽象化は低頻度バッチ処理に回せる
- 理論層を安定させつつ下位層を柔軟に保てる

### 実装含意

- MVP-1 は単層でもよい
- MVP-2 以降で層分離を視野に入れる
- 睡眠処理やオフライン圧縮との相性が良い

---

## 15. ハードウェア観点

RISA は GPU 最適というより、

- Graph Memory
- Graph Search
- Event Router
- Structure Processor

のような専用構成に向く可能性があります。

これは直ちに実装する話ではありませんが、RISA の推論速度問題を「GPU が遅いから不利」とだけ見るのは不十分です。

問題の本質は、計算資源ではなく **探索原理** にあります。

---

## 16. 実装への含意

このメモから、近い将来の実装で優先すべきものは次です。

### 16.1 MVP-1 以降で優先

- 活性化領域の上限制御
- ノード型・文脈・機能による簡易インデックス
- 頻出経路のショートカット
- 休眠化フラグ
- 抽象化のオフライン処理

### 16.2 研究課題として重要

- Relation Attention の定義
- 波的探索モデル
- 発達段階制御
- 多層記憶・多層推論

---

## 17. 結論

RISA の弱点である事前学習と推論速度は、構造編集そのものよりも、

> どこだけを見るか
> どこだけ起動するか
> どこだけ編集するか

を決める探索原理がまだ弱いことに由来している可能性が高いです。

したがって、RISA の実用化にとって重要なのは、Backpropagation に相当する学習則よりも、むしろ Attention に相当する **局所活性化・探索制御・関係選択の原理** を見つけることかもしれません。
