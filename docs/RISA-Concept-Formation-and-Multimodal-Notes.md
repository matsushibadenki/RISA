# RISA 概念形成とマルチモーダル学習メモ

## 1. このメモの目的

この文書は、RISA における概念形成、構造一致、状態遷移ベースの学習、そして映像・音への拡張方針を整理するための設計メモです。

MVP-1 技術設計書が「まず何を作るか」を固定する文書であるのに対して、このメモは「なぜその設計にするのか」「次にどこへ伸ばすのか」を残すことを目的とします。

---

## 2. RISA は拡張 RDB なのか

RISA は一見すると、リレーショナルデータベースや知識グラフを高度化したものに見えます。実際、ノード、関係、属性、履歴を持つ以上、表面的には近い部分があります。

しかし、RISA が本質的に目指しているのは、保存や検索の最適化ではありません。

RDB 的な系:

- 人間がスキーマを定義する
- 人間が関係型を決める
- 人間が概念境界を与える
- システムは整合的に保存・検索する

RISA 的な系:

- 観測から構造候補を自分で作る
- 共通性から中間概念を作る
- 例外を受けて概念を分裂させる
- 圧縮と予測性能を基準に構造を再編する

つまり違いは、データ構造そのものではなく、

> 構造を誰が作るか

にあります。

RISA が成立するなら、それは「高機能な保存系」ではなく、**構造仮説を自己生成し続ける動的システム**です。

---

## 3. ラベルだけでは概念は生まれない

`犬`, `猫`, `馬` というラベルだけから `哺乳類` は導けません。

```text
犬
猫
馬
```

だけでは、3 つの間の共通性に関する情報が存在しないからです。

RISA が上位概念を生成するには、少なくとも以下のいずれかが必要です。

- 共有属性
- 共有状態遷移
- 共有因果役割

例えば次のような構造があれば、共通部分を見つけられます。

```text
犬
・毛がある
・乳を飲む
・4本足
・走る
・呼吸する

猫
・毛がある
・乳を飲む
・4本足
・走る
・呼吸する

馬
・毛がある
・乳を飲む
・4本足
・走る
・呼吸する
```

このとき初めて、RISA は共通構造を抽出し、上位概念候補を作れます。

---

## 4. RISA が見るべき共通性

RISA の概念形成は、単なる属性クラスタリングでは不十分です。最低でも、次の 3 種類の共通性を扱う必要があります。

### 4.1 属性の共通性

静的特徴の一致です。

例:

- 毛がある
- 乳を飲む
- 肺呼吸する

### 4.2 状態遷移の共通性

変化パターンの一致です。

例:

```text
空腹
  ->
食べる
  ->
満腹
```

```text
疲労
  ->
休む
  ->
回復
```

### 4.3 因果役割の共通性

世界の中で果たす機能的位置の一致です。

例:

- 刺激に反応する
- 栄養を摂取する
- 疲労を蓄積する
- 休息で回復する

RISA の強みは、見た目や属性が異なっていても、状態遷移や因果役割が近ければ共通構造として扱えることにあります。

---

## 5. クラスタリングではなく構造一致

RISA が中心に置くべきなのは、距離ベースの分類ではなく **構造一致** です。

犬:

```text
毛
四足
乳
呼吸
走る
```

イルカ:

```text
乳
泳ぐ
肺呼吸
社会性
```

単純距離では遠く見えても、`乳` や `肺呼吸` のような重要構造は一致しているかもしれません。

つまり RISA が評価すべきなのは、

> 何が似ているか

よりも、

> どの構造が再利用可能か

です。

将来的には、以下のような一致関数が必要になります。

- 共有ノード比率
- 共有遷移比率
- 共有因果方向
- 文脈一致率
- 圧縮利得
- 予測改善量

---

## 6. 概念生成条件

RISA を単なる圧縮器にしないために、概念生成には複数条件を置くべきです。

### 6.1 必須条件

- 複数対象にまたがって同型の構造が反復する
- 共通構造をまとめると記述長が減る
- 新概念を導入すると予測性能が改善する
- 例外が少ない、または文脈分岐で扱える

### 6.2 圧縮だけでは足りない理由

圧縮だけを見ると、RISA は単なる要約器になります。

しかし本当に必要なのは、

> その概念を導入した結果、未来の状態遷移をよりうまく予測できるか

です。

この観点があることで、概念は単なる名前付きグループではなく、予測に使える再利用構造になります。

---

## 7. 概念は仮説である

RISA における概念は、固定的な真理ではなく **仮説的構造** です。

例えば `Concept001` は、後の経験によって次のように変化しえます。

- 分裂する
- 別概念と統合される
- 特定文脈へ限定される
- 不要なら消える

例:

```text
鳥 -> 飛ぶ
```

という概念が成立していても、

```text
ペンギン -> 飛ばない
```

が継続して観測されれば、

```text
鳥 --通常文脈--> 飛ぶ
鳥 --例外文脈--> 飛ばない
```

のように構造を再編成すべきです。

この可変性が、RISA を固定スキーマ的な DB から切り離す重要な特徴です。

---

## 8. 最小単位はノードではなく状態遷移かもしれない

RISA を深く考えると、最小単位はノードやラベルそのものではなく、**状態遷移** である可能性があります。

例えば「犬を知る」とは、名詞を覚えることではなく、

- 犬は走る
- 犬は疲れる
- 犬は食べる
- 犬は休むと回復する

のような変化パターンを理解することに近いかもしれません。

この立場では、RISA の最小表現は次です。

```text
State1
  ->
Event
  ->
State2
```

例:

```text
空腹
  ->
食べる
  ->
満腹
```

```text
疲労
  ->
休む
  ->
回復
```

このような状態遷移が反復し、圧縮されることで、

- 摂食
- 回復
- 運動
- 生存行動

のような概念が生まれると考えられます。

ここから導かれる重要な整理は次です。

> 概念とは、再利用可能な状態遷移の圧縮表現である

この観点に立つと、RISA はエンティティ中心の保存系ではなく、**状態遷移が自己組織化する系**として理解できます。

---

## 9. 映像と音に対する立場

映像や音を考えるとき、RISA は生の画素列や波形列そのものを直接処理する主戦場に立つ必要はありません。

むしろ RISA は、前段の知覚器によって抽出された

- 出現
- 消失
- 接触
- 分離
- 変位
- 強度変化
- 周波数変化
- 同時発生
- 遅延反応

といった **状態変化イベント** を扱う方が自然です。

したがって、当面の整理としては以下が妥当です。

- 知覚器: 生データからイベント候補を抽出する
- RISA: 抽出されたイベント列を統合し、概念形成・予測・説明を行う

---

## 10. 映像学習のアイデア

RISA にとって映像は「画像の集合」ではなく、**状態変化列** として扱うべきです。

例:

```text
ボールが落ちる
  ->
地面に接触する
  ->
跳ね返る
```

```text
人が手を伸ばす
  ->
コップをつかむ
  ->
持ち上げる
```

このとき RISA が保持すべきなのは、見た目そのものではなく、次のような変数です。

- 物体の出現
- 位置変化
- 接触
- 分離
- 速度変化
- 姿勢変化
- 状態遷移の順序

例えば、

```text
state: cup_on_table
  ->
event: hand_grasp
  ->
state: cup_in_hand
```

のような形式に変換できれば、映像も RISA の内部表現へ自然に落とせます。

### 10.1 映像における概念形成

映像では見た目の類似よりも、

- 同じ遷移パターンを持つか
- 同じ因果系列を持つか
- 同じ相互作用役割を持つか

が重要です。

例:

- コップを落とす
- ボールを落とす
- 皿を落とす

は見た目は違っても、

```text
支持喪失
  ->
落下
  ->
衝突
```

という構造は共通しています。

このような共通遷移を圧縮できれば、`落下` や `衝突` のような概念が作れます。

---

## 11. 音学習のアイデア

音も同様に、RISA にとって重要なのは波形の細部より **時間的変化構造** です。

音から抽出すべき要素の例:

- 発音開始
- 持続
- 強度変化
- 周波数変化
- リズム反復
- 無音区間
- 応答関係

例えば、

```text
音A
  ->
音B
  ->
相手が振り向く
```

が何度も起きれば、そこから `呼びかけ` に相当する機能概念が生まれる可能性があります。

また、

```text
泣き声
  ->
保護者が近づく
```

```text
警告音
  ->
回避行動
```

のように、音は状態変化を引き起こす因果トリガーとして学習できます。

### 11.1 音に対する RISA の適性

RISA は音分類器そのものというより、

- どの音が何の前触れか
- どの音がどんな行動を誘発するか
- どの音がどの文脈で意味を変えるか

を構造的に表す **音の意味作用モデル** に向いています。

---

## 12. 映像と音の統合

映像と音は別々の世界ではなく、同一世界の複数観測面です。RISA の理想は、それらを同じ状態遷移基盤へ統一することです。

例:

```text
映像: コップが落ちる
音: ガシャン
状態: 破損
```

これが繰り返されれば、

```text
落下
  ->
衝突音
  ->
破損
```

というマルチモーダルな因果構造が作れます。

また、

```text
映像: 口が動く
音: 発話が出る
相手: 振り向く
```

のような系列からは、コミュニケーション行為の原型を学べる可能性があります。

最終的には、

- 出現
- 変化
- 接触
- 反応
- 因果
- 予兆
- 同時発生
- 遅延発生

を、視覚・聴覚・行動の区別を越えて同じ形式で扱うことが理想です。

## 13. 実装上の含意

この議論から、実装上の重要な方向性がいくつか導かれます。

### 13.1 MVP-1

- 構造化イベント入力に限定する
- 状態遷移の圧縮と予測をまず成立させる
- 知覚は外部に置く

### 13.2 MVP-2 以降

- 文脈分裂
- 矛盾管理
- 睡眠処理による再編
- 知覚器からのイベント抽出連携

### 13.3 将来のマルチモーダル化

- 視覚イベント抽出器
- 音イベント抽出器
- 複数モダリティの時系列整列
- 同時発生と遅延因果の学習

---

## 14. 今後評価すべきこと

RISA の評価は、単なる分類精度では不十分です。今後は少なくとも以下を重視すべきです。

- どれだけ少ない構造で多くの経験を説明できたか
- どれだけ少ない概念で多くの未来遷移を予測できたか
- 新しい観測を既存構造を壊さず取り込めたか
- 例外を削除ではなく文脈として扱えたか
- 映像・音・行動の間の共通遷移構造を見つけられたか

最も重要な評価軸は、次の一文で表せます。

> どれだけ少ない再利用可能構造で、どれだけ多くの状態遷移を予測できたか

---

## 15. 現時点の結論

現時点では、RISA を単なる賢いデータベースとして捉えるよりも、

> 状態遷移が自己組織化し、その圧縮構造から概念・予測・説明を生み出す動的システム

として定義する方が筋が良いです。

この立場をとるなら、映像も音も、そのままの生信号ではなく、状態変化と相互作用の連鎖として取り込み、同一の世界モデルへ統合する方向が自然です。

MVP-1 ではそこまで実装しないとしても、将来の設計原則として保持しておく価値があります。
