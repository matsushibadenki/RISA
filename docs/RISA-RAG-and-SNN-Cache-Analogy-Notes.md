# RISA と RAG、および SNN のキャッシュ類比メモ

## 1. 目的

この文書は、次の問いを整理するための設計メモです。

- RISA における SNN は RAG のようなものか
- RAG と RISA はどこが似ていて、どこが違うのか
- SNN を RISA の検索器と見るべきか、それとも別のものと見るべきか

結論を先に言うと、

> RISA における SNN は RAG にかなり近い発想を持つが、本質は違う

です。

さらに厳密に言うと、RISA における SNN は RAG よりも **OS のキャッシュ層** に近い可能性があります。

---

## 2. RAG と RISA の比較

まず、RAG と RISA を並べると次のようになります。

| 項目 | RAG | RISA |
| --- | --- | --- |
| 外部知識 | 取得する | 自分で育てる |
| 検索 | ベクトル検索 | 構造探索 |
| 更新 | 基本しない | 常に更新する |
| 記憶 | 文書 | 世界モデル |
| 推論 | LLM が担当 | RISA 自身が担当 |

この比較から分かる最大の違いは、知識の所在です。

### RAG

```text
質問
  ->
検索
  ->
外部文書取得
  ->
LLM
```

知識は外にあります。  
LLM はそれを読んでいるだけです。

### RISA

```text
経験
  ->
世界モデル更新
  ->
推論
```

知識は外部文書ではなく、**RISA 自身の内部世界モデル** です。

---

## 3. どこが似ているのか

RISA における SNN 的な役割が RAG に近く見えるのは、**必要な関連構造をその都度取り出す** からです。

例えば RISA に

```text
犬
```

が来ると、

```text
犬
  ->
哺乳類
  ->
運動
  ->
疲労
```

のような関連構造が局所的に活性化されます。

これは確かに、

```text
質問
  ->
検索
  ->
関連文書取得
```

という RAG 的な流れに似ています。

つまり共通する発想は、

> 全部を毎回使うのではなく、その時必要なものだけ取り出す

という点です。

---

## 4. どこが本質的に違うのか

しかし、本質的な違いはその後にあります。

RAG は検索して終わりです。  
取得結果は LLM の入力になるだけで、検索器自身は基本的に変化しません。

一方 RISA では、検索された構造そのものが次の学習で変化します。

例えば、

```text
犬
  ->
運動
  ->
疲労
```

を何度も使えば、

```text
犬
  =>
疲労
```

のようなショートカットや圧縮構造が生まれるかもしれません。

つまり、

> RISA は検索結果によって自分自身が進化する

のに対し、

> RAG は検索結果を読むだけで、自分自身の知識構造は育たない

のです。

---

## 5. SNN は RAG よりキャッシュに近い

この違いを踏まえると、RISA における SNN 的役割は、RAG というより **OS のキャッシュ** に近いと考えられます。

CPU は毎回すべてのメモリを見ません。

```text
L1
L2
L3
```

のようなキャッシュ層を使います。

同様に、RISA が 100 億ノード規模の世界モデルを持つとしても、実際の推論で毎回全体を使うわけではありません。

```text
RISA
100億ノード
  ->
SNN / 活性化層
今必要な100ノードだけ
```

のように、その時点で必要な近傍だけを前面に出す方が自然です。

この意味で SNN は、

- 検索器
- 局所活性化器
- キャッシュ層

の三つを部分的に兼ねる可能性があります。

---

## 6. RAG とキャッシュの違い

ここでさらに整理すると、RAG とキャッシュも厳密には違います。

### RAG 的発想

- 外部知識ベースから関連情報を持ってくる
- 元のモデルはその知識を内在化していない

### キャッシュ的発想

- もともと内部にある巨大知識のうち、最近・頻繁に使うものだけ前面に置く

RISA における SNN 的層は、後者に近いです。

つまり、

> 外から読むのではなく、中にある世界モデルの一部を素早く再活性化する

のが本質です。

---

## 7. RISA の検索は retrieval ではなく reactivation かもしれない

この観点から見ると、RISA の局所探索は単なる retrieval ではなく、

> reactivation

と呼ぶ方が近いかもしれません。

つまり、

- 文書を取りに行くのではない
- 既に育っている構造の一部を再活性化する

のです。

この差は重要です。

RAG では、知識は常に「外部のどこかにある」。  
RISA では、知識は「内部構造としてすでに存在し、その一部が光る」。

---

## 8. 実装への含意

この整理から、RISA 実装では次の方向が重要になります。

### 8.1 外部検索を中心に据えない

RISA の核心は外部検索統合ではなく、内部世界モデルの構造更新です。

### 8.2 局所活性化層を設ける

巨大な世界モデルの全域ではなく、最近使った概念や現在文脈に近い構造だけを前面に出す層が必要です。

### 8.3 キャッシュが構造進化に影響するようにする

単に高速化するだけでなく、

- 何度も使われる経路を強化する
- 近道を作る
- 抽象化候補を昇格させる

など、キャッシュ利用が構造代謝へつながる必要があります。

---

## 9. SNN の役割再定義

RISA 文脈で SNN を使うなら、その役割は次のように再定義できます。

- 時間順序を統合する
- 局所的な関連構造を起動する
- 最近使われた構造を前面化する
- RISA 全体の即時推論用キャッシュとして働く

この意味で SNN は、

> RAG 的 retrieval 層というより、RISA の動的活性キャッシュ層

と呼ぶ方が適切かもしれません。

---

## 10. 結論

RISA における SNN は、一見すると RAG に似ています。

どちらも、

> 必要な関連情報だけをその都度取り出す

という発想を持っているからです。

しかし本質的には、

- RAG は外部知識を読む
- RISA は内部世界モデルを再活性化する
- RISA は検索の結果によって自分自身を再編成する

点で異なります。

そのため、RISA における SNN は、RAG の検索器というよりも、

> 内部世界モデルの一部を高速に前面化するキャッシュ兼局所活性化層

として理解する方が、設計上かなり筋が良いです。
