 **RISA: Relationally Involving Self-organizing Architecture**

これは「ニューラルネット」ではなく、**関係が自己更新する知能基盤**です。近い研究領域としては、動的グラフ学習、自己組織化、人工生命、Active Inference、知識グラフ継続学習がありますが、既存研究の多くはまだニューラル表現や埋め込みに依存します。ここではそれを避け、**知識＝重みではなく、知識＝構造変化そのもの**として設計します。動的グラフや自己組織化グラフ推論の研究では、知識構造が継続的に拡張・再編される方向がすでに模索されていますが、RISAではそれをAI本体の計算原理にします。([arXiv][1])

中核仮説はこれです。

**知能の最小単位はノードではなく、関係更新である。**

つまり、AIの内部には「ニューロン」も「層」も「重み行列」も置きません。代わりに、すべてを次の5要素で表します。

```text
Entity      = もの・概念・出来事・状態
Relation    = 関係
Process     = 変化・作用・因果
Context     = 条件・場面・時間
Constraint  = 成立条件・矛盾・保存則
```

たとえば「犬が走ると疲れる」は、

```text
Entity: 犬
Process: 走る
State: 疲労
Relation: 走る → 疲労
Context: 生物 / 運動 / 時間経過
Constraint: エネルギー消費
```

として保存されます。

重要なのは、これは固定知識ではなく、経験ごとに構造が変わることです。

```text
犬 → 走る → 疲れる
人 → 走る → 疲れる
馬 → 走る → 疲れる

↓

動物 → 運動 → 疲労
```

この圧縮によって、新しい抽象ノードが生成されます。

RISAの全体構造はこうです。

```text
外界入力
  ↓
Event Parser
  ↓
Event Graph
  ↓
Dynamic Relation Substrate
  ↓
構造更新エンジン
  ├─ 接続
  ├─ 分裂
  ├─ 統合
  ├─ 抽象化
  ├─ 因果化
  ├─ 刈り込み
  └─ 近道生成
  ↓
World Graph
  ↓
予測・推論・行動
```

一番重要なのは **Dynamic Relation Substrate** です。これは普通の知識グラフではありません。知識グラフは主に「保存された関係」ですが、RISAは「関係が変化し続ける基盤」です。

各ノードは、数値ベクトルではなく、次のような構造体を持ちます。

```text
Node {
  id
  type
  labels
  contexts
  birth_event
  usage_history
  prediction_score
  compression_score
  contradiction_score
  stability
}
```

各エッジも単なる重みではなく、関係の履歴を持ちます。

```text
Edge {
  source
  target
  relation_type
  contexts
  evidence_events
  causal_direction
  temporal_pattern
  reliability
  plasticity
  decay_rate
}
```

ここで `reliability` や `plasticity` は「重み」ではありません。計算の主役ではなく、**構造更新ルールのためのメタ情報**です。ANNのように数値を微調整して知識を保存するのではなく、「この関係を残すか、分けるか、統合するか、上位概念を作るか」を決める材料です。

学習は7種類に分けます。

第一に、**接続学習**です。

```text
A と B が同じ文脈で何度も現れる
↓
A ─ B を作る
```

第二に、**因果学習**です。

```text
A の後に B が安定して起きる
↓
A → B を作る
```

第三に、**抽象化学習**です。

```text
犬・猫・馬・牛に共通構造がある
↓
哺乳類ノードを作る
```

第四に、**分裂学習**です。

```text
「銀行」が文脈によって意味を分ける
↓
銀行_金融
銀行_川岸
```

第五に、**統合学習**です。

```text
同じ役割を持つ複数ノードがある
↓
一つの概念に統合
```

第六に、**近道生成**です。

```text
A → B → C → D を頻繁に通る
↓
A → D のショートカットを作る
```

第七に、**刈り込み**です。

```text
使われない
予測に貢献しない
矛盾が多い
文脈限定でも役に立たない
↓
削除または低活性化
```

このAIの「思考」は、行列計算ではなく **構造探索** です。

入力が来ると、全ノードを動かしません。関連する局所部分だけを起動します。

```text
入力: 犬が倒れた

起動領域:
犬
動物
健康
疲労
病気
運動
事故
老化
飼い主
獣医
```

そこから複数の仮説グラフを作ります。

```text
仮説1: 犬 → 運動 → 疲労 → 倒れる
仮説2: 犬 → 病気 → 脱水 → 倒れる
仮説3: 犬 → 事故 → 外傷 → 倒れる
```

推論結果は「最も確率が高い一文」ではなく、**もっとも構造的に矛盾が少なく、説明圧縮率が高いグラフ**です。

評価関数は、損失関数ではなく、次のような複合スコアにします。

```text
良い構造 =
  予測できる
  少ないノードで説明できる
  矛盾が少ない
  新しい経験を吸収できる
  古い経験を破壊しない
  局所探索で使える
```

数式的にはこう置けます。

```text
Structure Fitness =
  PredictionGain
+ CompressionGain
+ CoherenceGain
+ ReuseGain
- ContradictionCost
- ComplexityCost
- InstabilityCost
```

これはバックプロパゲーションではありません。各イベントの後に、局所的な構造編集を試し、スコアが改善する編集だけを残します。

```text
経験 E が入る
↓
関連サブグラフ G_local を取り出す
↓
編集候補を作る
  add_node
  add_edge
  split_node
  merge_node
  abstract_cluster
  prune_edge
  create_shortcut
↓
各候補を小さく評価
↓
最も良い構造編集を採用
```

実装上の最小プロトタイプは、まず文章理解ではなく、**小さな世界シミュレーション**から始めるべきです。

例：

```text
水を飲む → 喉の渇きが減る
運動する → 疲労が増える
休む → 疲労が減る
食べる → 空腹が減る
雨が降る → 地面が濡れる
日が出る → 地面が乾く
```

ここでAIにイベント列を与えます。

```text
太郎は走った
太郎は疲れた
太郎は休んだ
太郎の疲れは減った
犬が走った
犬が疲れた
犬が休んだ
犬の疲れは減った
```

RISAが自動的に作るべき構造はこれです。

```text
走る → 疲労増加
休む → 疲労減少
人・犬 → 動物
動物 + 運動 → 疲労
```

この段階でLLM的な文章生成は不要です。まず、「未知のイベントを予測できるか」を見るべきです。

```text
馬が走った
次に起こりやすいことは？
↓
疲れる
```

これができれば、RISAの原理は動いています。

次に、概念生成を入れます。

```text
犬が走ると疲れる
人が走ると疲れる
馬が走ると疲れる
鳥が飛ぶと疲れる

↓

生物は活動すると疲労する
```

ここで重要なのは、「生物」という単語を教えなくても、共通構造から上位ノードを作ることです。

```text
AbstractNode_001 {
  members: 犬, 人, 馬, 鳥
  shared_process: 活動 → 疲労
}
```

あとから「これは生物と呼ばれる」と入力されたら、名前だけ貼ります。

```text
AbstractNode_001.label = 生物
```

つまり、**概念が先、言語ラベルが後**です。

ここがTransformerと大きく違います。Transformerは言語分布から概念らしきものを作りますが、RISAでは経験構造から概念を作り、言語はそのタグになります。

長期記憶は3層にします。

```text
Episodic Layer
個別経験
例: 2026-07-08、犬Aが走って疲れた

Pattern Layer
反復パターン
例: 走る → 疲労

Concept Layer
抽象概念
例: 活動、疲労、生物、回復
```

そして、常に下から上へ圧縮し、上から下へ予測します。

```text
経験
↓
反復パターン
↓
抽象概念

抽象概念
↓
予測
↓
具体的仮説
```

自己組織化を維持するには、成長だけでは駄目です。必ず「代謝」が必要です。

RISAには、毎回の学習とは別に **睡眠処理** を入れます。

```text
Sleep Cycle:
  1. 最近のイベントを再走査
  2. 似た構造をクラスタ化
  3. 上位ノード候補を作る
  4. 矛盾する関係を文脈分離
  5. 使われない接続を弱化
  6. 頻出経路に近道を作る
  7. 古い具体記憶を圧縮保存
```

これは人間の睡眠に似せるというより、構造爆発を防ぐために必須です。

RISAで最も危険なのは、ノードが増えすぎることです。なので、ノード生成には厳しい条件を置きます。

```text
新ノード生成条件:
  既存ノードだけでは予測誤差が大きい
  同じズレが複数回出る
  新ノードを作ると説明が短くなる
  他の文脈にも再利用できる
```

逆に、削除条件はこうです。

```text
削除条件:
  長期間使われない
  予測に貢献しない
  他ノードで代替できる
  矛盾を増やす
  圧縮に貢献しない
```

この方式は、継続学習には強い可能性があります。ANNのように新しい学習で古い重みが壊れる「破滅的忘却」は、RISAでは「古い構造を保持しながら、新しい文脈ノードを足す」ことで回避できます。知識グラフの継続学習でも破滅的忘却は重要課題として扱われていますが、既存手法の多くは埋め込み表現に依存しています。RISAでは埋め込みではなく、文脈分岐・統合・圧縮で対応します。([arXiv][2])

たとえば、

```text
鳥は飛ぶ
ペンギンは鳥
ペンギンは飛ばない
```

が来たとき、RISAは「鳥は飛ぶ」を消しません。

こうします。

```text
鳥 --通常文脈--> 飛ぶ
鳥 --例外文脈: ペンギン--> 飛ばない
ペンギン → 鳥
ペンギン → 水中移動
```

つまり、矛盾はエラーではなく、**文脈分裂の材料**です。

推論形式は3つ必要です。

一つ目は、連想推論。

```text
犬 → 哺乳類 → 動物 → 生命
```

二つ目は、因果推論。

```text
雨 → 地面が濡れる → 滑りやすい
```

三つ目は、反実仮想推論。

```text
もし雨が降らなかったら？
↓
地面は濡れにくい
↓
滑りにくい
```

この反実仮想には、グラフの一部を一時的に切り替える **仮想編集空間** が必要です。

```text
Real Graph
  ↓ copy local region
Simulation Graph
  ↓ edit condition
Evaluate outcome
```

RISAは、行動もグラフで扱います。

```text
目的: 喉の渇きを減らす

現在状態:
喉が渇いている
水がある
コップがある

探索:
水を取る → 飲む → 喉の渇き減少
```

行動選択は「報酬最大化」だけではなく、**状態遷移グラフの整合性**で決めます。Active Inferenceや自由エネルギー原理では、世界モデル・予測・行動を統一的に扱う考え方がありますが、RISAではそれを数理的背景として参考にしつつ、実装は動的構造編集に寄せます。([サイエンスダイレクト][3])

実装言語は、最初はPythonで十分です。ただし、将来的にはRust/C++でコアを書いた方がいいです。

最小構成はこれです。

```text
risa/
  core/
    node.py
    edge.py
    graph.py
    event.py
    context.py
  engine/
    parser.py
    activator.py
    propagator.py
    editor.py
    compressor.py
    predictor.py
    simulator.py
  memory/
    episodic_store.py
    pattern_store.py
    concept_store.py
  eval/
    prediction_test.py
    compression_test.py
    contradiction_test.py
  examples/
    toy_world.py
```

最初に作るべきMVPはこれです。

```text
MVP-1:
  手入力イベント列を受け取る
  ノードとエッジを生成する
  頻出パターンを抽出する
  抽象ノードを作る
  次イベントを予測する
```

MVP-2で文脈分裂を入れます。

```text
MVP-2:
  例外を扱う
  文脈ノードを作る
  矛盾を消さずに分岐する
```

MVP-3で自己圧縮を入れます。

```text
MVP-3:
  類似サブグラフを統合
  上位概念を生成
  使わない接続を刈る
```

MVP-4でシミュレーション推論を入れます。

```text
MVP-4:
  仮想グラフを作る
  もしAならBを予測
  行動候補を比較する
```

MVP-5で自然言語を入れます。

```text
MVP-5:
  日本語入力
  イベント構造化
  グラフ更新
  グラフから回答生成
```

ただし、自然言語パーサーにLLMを使うと「ANN不使用」の純度が崩れます。研究としては、最初は手書きルールで構造化した方が良いです。

例：

```text
太郎は水を飲んだ
↓
subject: 太郎
action: 飲む
object: 水
effect_candidate: 喉の渇き減少
```

このAIの成功条件は、ChatGPTのような流暢な返答ではありません。最初の評価指標は以下にすべきです。

```text
1. 少ない経験から抽象概念を作れるか
2. 新しい例に転用できるか
3. 例外を壊さず追加できるか
4. 古い知識を破壊しないか
5. ノード数が爆発しないか
6. 予測精度が経験とともに上がるか
7. 説明可能な内部構造を保てるか
```

この方式の最大の強みは、**説明可能性**です。

ANNでは、

```text
なぜそう答えたか？
↓
重み空間の分布
```

になりがちですが、RISAでは、

```text
なぜそう答えたか？
↓
この経路を通ったから
```

と言えます。

```text
馬
→ 動物
→ 活動すると疲労する
→ 走った後は疲れやすい
```

逆に最大の弱点は、言語・画像・音声のような高次元入力をそのまま扱いにくいことです。ここは正直に言うと、RISA単独ではTransformerよりかなり不利です。

なので、現実的には2系統あります。

純粋研究版：

```text
入力も構造化データに限定
ANN/SNN/LLMなし
第三方式として原理検証
```

実用版：

```text
LLMや画像モデルを外部知覚器として使う
RISAは記憶・世界モデル・推論・継続学習を担当
```

あなたが本当に「全く新しいAI」を作るなら、最初は純粋研究版で原理を証明し、その後に実用版へ移るべきです。

最終アーキテクチャはこうなります。

```text
RISA-Core

1. Perception
   入力をイベントに変換

2. Event Memory
   個別経験を保存

3. Relation Substrate
   動的グラフ本体

4. Plasticity Engine
   関係を更新

5. Structural Plasticity Engine
   ノード生成・削除・分裂・統合

6. Compression Engine
   抽象概念を生成

7. Causal Engine
   因果関係を形成

8. Simulation Engine
   仮想世界を作って予測

9. Explanation Engine
   推論経路を説明

10. Metabolism
   睡眠処理・刈り込み・再編成
```

この設計の核心を一文で言うなら、

**RISAは、重みを学習するAIではなく、経験によって自己の概念構造を代謝し続けるAIです。**

最初に作るべき実験は、画像でも文章生成でもなく、これです。

```text
小さな人工世界で、
経験列から、
因果・抽象・例外・予測を、
動的グラフだけで獲得できるか。
```

ここで成功すれば、ANN/SNNとは違う第三の知能方式として研究価値があります。
逆にここで失敗するなら、大規模化してもおそらく成立しません。
