# RISA と ANN / SNN の比較評価メモ

## 1. 目的

この文書は、RISA を ANN や SNN と比較したときに、アーキテクチャ上どのような長所・短所が予想されるかを整理するための設計メモです。

ここで述べる内容は、現時点ではすべて **実装前の予想** です。断定ではなく、今後の実験で検証すべき仮説として扱います。

---

## 2. 比較の観点

RISA を比較するときは、「何を学習対象としているか」で見るのが分かりやすいです。

| 項目 | ANN | SNN | RISA |
| --- | --- | --- | --- |
| 学習対象 | 重み | 重み + 時間 | 構造 |
| 記憶 | 分散表現 | 分散表現 + 時間 | 明示構造 |
| 推論 | 行列演算 | イベント伝播 | グラフ探索 / 構造変形 |
| 学習 | 数値最適化 | 可塑性更新 | 構造編集 |

この違いから、RISA の利点と弱点はかなりはっきり予想できます。

---

## 3. 大規模事前学習

### 3.1 ANN

ANN、特に Transformer は大規模事前学習に非常に強いです。

理由:

- 同一形の計算を大量並列化しやすい
- GPU と相性が良い
- 巨大データセットの反復学習に向く

### 3.2 SNN

SNN は現時点では大規模学習アルゴリズムがまだ発展途上です。

### 3.3 RISA

RISA は、現時点の予想では大規模事前学習はかなり不利です。

理由:

- 追加
- 削除
- 統合
- 分裂
- 文脈分岐

のような構造編集が大量に発生するためです。

これは GPU が最も得意とする一様な行列演算とは逆方向の計算です。

### 3.4 暫定評価

- ANN: 非常に強い
- SNN: まだ弱め
- RISA: 弱い可能性が高い

現時点の予想では、RISA の大規模事前学習能力は `★☆☆☆☆〜★★☆☆☆` 程度です。

---

## 4. 継続学習と事後学習

ここは RISA が最も有利になる可能性がある領域です。

### 4.1 ANN の弱点

ANN は、

```text
新しい知識
  ->
重み更新
  ->
古い知識が壊れる
```

という形で、破滅的忘却が起こりやすいです。

### 4.2 RISA の利点

RISA では、

```text
新しい知識
  ->
新ノードや新関係の追加
  ->
既存構造を維持したまま拡張
```

が可能です。

このため、継続学習と事後学習では非常に有利かもしれません。

### 4.3 暫定評価

- ANN: 弱い
- SNN: 比較的強い
- RISA: 非常に強い可能性

現時点では `★★★★★` を狙える領域です。

---

## 5. 説明可能性

### 5.1 ANN

ANN では「なぜその答えになったか」が重み空間へ分散して埋め込まれるため、説明が難しくなりがちです。

### 5.2 SNN

SNN は発火系列やタイミングを追うことで ANN よりは説明しやすい可能性がありますが、依然として複雑です。

### 5.3 RISA

RISA では原理的に

```text
なぜ？
  ->
この構造を通ったから
```

と返しやすいです。

つまり、

- どのイベントが根拠か
- どの概念を経由したか
- どの文脈が効いたか

を明示しやすい。

### 5.4 暫定評価

RISA の説明可能性は非常に強い可能性があります。  
現時点では `★★★★★` を期待できます。

---

## 6. 推論速度

ここは単純に有利とも不利とも言い切れません。

### 6.1 不利な面

- 大規模グラフ探索は重い
- ノード数と関係数が増えるほど候補比較が増える
- 構造編集まで含めると計算コストが高い

### 6.2 有利な面

- 全体を毎回計算する必要はない
- 局所探索だけで済めば軽い
- 適切な活性化制御があれば省計算化できる

### 6.3 暫定評価

RISA の推論速度は、探索アルゴリズム次第です。

予想としては、

- Transformer よりは遅い
- うまく局所化できれば巨大グラフ全探索よりはかなり速い

で、現時点では `★★★☆☆` 程度と考えるのが妥当です。

---

## 7. メモリ効率

ここは現時点で最も不確実性が高い項目です。

### 7.1 不利な可能性

RISA は知識をノード、エッジ、履歴、文脈として明示的に持つため、初期段階ではかなり巨大化する可能性があります。

### 7.2 有利な可能性

圧縮と抽象化が十分うまく機能すれば、

```text
多数の具体例
  ->
少数の再利用可能概念
```

へ落とせるため、長期的には効率化できる可能性があります。

### 7.3 暫定評価

RISA のメモリ効率は、**圧縮能力次第** です。  
この点は実装検証が必要です。

---

## 8. 抽象概念生成

ここは RISA の最大の武器になりうる領域です。

ANN でも抽象化は起きていますが、多くは暗黙的です。

RISA の場合は、

```text
哺乳類
```

のような概念が実際の構造ノードとして生まれます。

つまり、

- 概念が見える
- 概念を編集できる
- 概念を分裂・統合できる
- 概念を説明に使える

という点が大きいです。

### 暫定評価

RISA の抽象概念生成能力は `★★★★★` を期待できる領域です。  
ただし、本当にそれが成立するには、概念生成条件と探索制御が成功する必要があります。

---

## 9. 少量学習

RISA は少量学習に強い可能性があります。

理由:

- 数万例を必要とする数値近似ではなく
- 少数の構造一致から共通概念を作れる可能性がある

例えば、

```text
dog run -> fatigue_up
human run -> fatigue_up
horse run -> fatigue_up
```

のような少数例でも、

```text
run -> fatigue_up
```

の共有構造を見つけられるなら、そこから一般化できます。

### 暫定評価

実際にうまくいけば `★★★★★` に近い領域です。

---

## 10. 忘却制御

RISA では忘却が「失敗」ではなく、「制御可能な構造代謝」になる可能性があります。

例えば、

```text
長期間使わない
  ->
弱化
  ->
圧縮保存
  ->
削除
```

のような操作を明示的に設計できます。

これは ANN ではやりにくい利点です。

### 暫定評価

- ANN: 苦手
- SNN: 比較的得意
- RISA: 制御しやすい可能性

RISA は `★★★★★` を狙える領域です。

---

## 11. エネルギー効率

RISA は全ノードを毎回計算せず、局所だけ活性化できれば、高効率になる可能性があります。

### 有利な条件

- 関連領域だけを起動する
- 同じ構造を再利用する
- 必要な局所探索だけに絞る

### 不利な条件

- 毎回探索が広がりすぎる
- 構造爆発で比較対象が多すぎる
- 編集コストが高すぎる

### 暫定評価

局所探索が本当に成立すれば `★★★★☆` もありえます。  
ただし探索設計に失敗すれば大きく悪化します。

---

## 12. ハードウェア適性

ANN は GPU と相性が良く、SNN はニューロモーフィックとの親和性が高いです。

RISA は、もし本格的に成立するなら、次のような専用設計が向く可能性があります。

- Graph Memory
- Event Router
- Structure Processor

つまり、RISA は GPU に最適化された方式ではなく、**グラフ探索と構造編集に特化したアーキテクチャ** が必要になるかもしれません。

これは将来的な研究課題です。

---

## 13. 創造性

RISA の創造性は、テキスト生成の流暢さではなく **新しい構造生成** として現れる可能性があります。

例えば、

- `哺乳類`
- `育児動物`
- `回復行動`
- `危険予兆`

のように、同じ経験集合から複数の観点で新しい概念を作れるかもしれません。

もしこれが成立するなら、RISA の創造性は「確率的補完」ではなく、

> 新しい再利用構造を発見すること

になります。

---

## 14. 最大の弱点

RISA の最大の弱点として、現時点では次の 3 つを強く警戒すべきです。

- 探索爆発
- 構造爆発
- 有意味な概念生成の難しさ

特に探索爆発は核心です。

例えば巨大グラフに対して、

```text
何を比較するか
どこを見るか
どの編集候補だけ試すか
```

を適切に絞れないと、RISA は動きません。

これは単なる最適化問題ではなく、方式の成立条件そのものです。

---

## 15. RISA の核心は学習より探索かもしれない

ANN では核心は最適化でした。  
SNN では時間と可塑性が大きなテーマです。

RISA では、もしかすると最重要なのは学習則そのものより、

> どこだけを見るか
> いつ構造編集を起こすか

を決める制御原理かもしれません。

言い換えると、RISA のブレークスルーはバックプロパゲーションの代替というより、

> バックプロパゲーションに匹敵するほど強力な構造探索・構造編集アルゴリズム

の発見にある可能性があります。

## 17. 実装への含意

この比較から、RISA の実装では少なくとも次を優先すべきです。

### 17.1 優先して研究すべきこと

- 局所探索の制御
- 編集候補生成の制限
- 構造圧縮
- 概念生成条件
- 継続学習時の安定性

### 17.2 安易に期待しすぎないこと

- いきなり大規模事前学習で ANN を超えること
- GPU 上で同じスケーリング則を持つこと
- 何もしなくても抽象概念がきれいに生まれること

### 17.3 早期に確認すべきこと

- 少量学習は本当に成立するか
- 継続学習で忘却が少ないか
- 抽象化が予測改善へつながるか
- 探索コストが許容範囲に収まるか

---

## 18. 結論

現時点の予想では、RISA は ANN のような大規模事前学習装置にはなりにくい一方で、

- 継続学習
- 説明可能性
- 明示的抽象化
- 少量学習
- 制御可能な忘却

では非常に強い可能性があります。

逆に成立を左右する最大の難所は、

> 構造探索と構造編集を、いつ、どこで、どれだけ行うか

という制御原理です。

この問題に十分強い答えが出せなければ、RISA は理論的に面白くても実用システムにはなれません。  
逆にここを突破できれば、ANN / SNN とはかなり違う第三の知能方式として成立する可能性があります。

補足:

ANN / SNN / RISA をどのように役割分担させるかについては、[RISA と Transformer / SNN の関係メモ](RISA-Transformer-SNN-Relationship-Notes.md) に整理します。現時点では、Transformer は知覚、SNN は時間統合、RISA は概念形成・世界モデルという補完関係が有力です。
