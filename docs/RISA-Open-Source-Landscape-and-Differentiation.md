# RISA と既存オープンソースの近接領域メモ

## 1. 目的

この文書は、RISA の設計思想に近い既存オープンソースや研究の位置関係を整理し、RISA の独自性を明確にするための設計メモです。

現時点の結論を先に言うと、次になります。

> RISA に部分的に近いプロジェクトは存在するが、Concept Cell の自己組織化から世界モデルが創発するところまで踏み込んだものは、少なくとも現時点ではほとんど見当たらない

したがって既存 OSS は、RISA の完全な代替ではなく、

- イベント駆動
- スティグマジー
- 世界モデル更新
- 知識グラフ表現

といった個別部品の参照元として扱うのがよい。

---

## 2. 現時点での近さの整理

会話ベースの整理では、次のような比較が妥当です。

| プロジェクト / 系譜 | RISA との近さ | 主に参考になる点 |
| --- | :---: | --- |
| ActiveGraph | ★★★★★ | イベント駆動、関係中心、エッジ側の振る舞い |
| GraphPalace | ★★★★☆ | スティグマジー、利用頻度で構造が変わる記憶 |
| Active Inference 系 | ★★★★☆ | 世界モデル更新、予測と観測の往復 |
| OriginTrail DKG | ★★★☆☆ | 永続知識グラフ、構造の共有と維持 |
| Agentic Deep Graph Reasoning 系 | ★★★☆☆ | 概念の連鎖的拡張、自己組織化的グラフ成長 |
| Atomic / 個人知識グラフ | ★★☆☆☆ | 概念整理、参照構造、再利用可能な知識単位 |
| Neo4j 系 Graph AI | ★★☆☆☆ | グラフ基盤、探索、可視化、問い合わせ |

重要なのは、これらの多くが

```text
Graph
  ->
更新
```

という発想に立っていることです。

RISA が本当に目指したいのは、むしろ

```text
Concept Cell
Concept Cell
Concept Cell
  ->
局所ルールで自己組織化
  ->
Graph が創発
```

という逆向きの発想です。

---

## 3. ActiveGraph から学べること

ActiveGraph 的な系は、現時点で最も RISA に近い参考例の一つです。

注目点:

- Event を中心に更新が進む
- Graph 更新が行動や後続処理を引き起こす
- 関係や依存の側に振る舞いを書き込める

これは RISA の

```text
犬
  ->
運動
  ->
疲労
```

のような関係鎖に、単なる保存だけでなく「意味のある反応」を持たせたいという発想と近い。

ただし、RISA が目指しているのはそこで止まりません。

不足している点:

- 概念が自律的に分裂・融合・休眠・死滅すること
- `哺乳類` のような圧縮概念が自己生成されること
- 構造編集主体が中央エンジンではなく Concept Cell であること

---

## 4. GraphPalace から学べること

GraphPalace 的な系で特に面白いのは、**使われるほど痕跡が強くなる** というスティグマジー型の記憶です。

これは RISA の次の設計と非常に相性が良いです。

- よく使われる経路を太くする
- 使われない経路を減衰させる
- 頻用パターンにショートカットを作る
- 探索を交通量ベースで局所化する

つまり、以前整理した

- 正のフィードバック
- 負のフィードバック
- 散逸構造としての情報流

の考え方を、実装レベルへ落とし込むヒントになります。

一方で、GraphPalace がそのまま RISA になるわけではありません。

RISA ではさらに、

- 利用による強化
- 制約による刈り込み
- 類似構造の融合
- 矛盾による分裂
- 圧縮利益による抽象化

まで含めた **構造代謝** が必要です。

---

## 5. Active Inference から学べること

Active Inference 系は、RISA とかなり相性が良いです。

共通する問題意識:

- 世界モデルを更新する
- 予測と観測の差から学ぶ
- 次に何を観測すべきかを決める

RISA の将来像として出てきた

- Transformer が仮説を出す
- RISA が世界モデルと照合する
- RISA が追加観測を要求する

というループは、Active Inference 的な見方と自然に接続できます。

ただし RISA の独自点は、世界モデルの担い手を固定グラフや固定パラメータではなく、**自己組織化する Concept Cell 群** と見なすところにあります。

---

## 6. LLM 駆動の自己組織化グラフ研究から学べること

LLM を使って概念ネットワークや知識グラフを自己拡張していく研究は、RISA にかなり近い問題を扱っています。

参考になる点:

- 概念から概念を生む再帰性
- グラフの成長を通じた知識拡張
- 単発推論ではなく、継続的な世界モデル更新

ただし、これらの多くはまだ

- LLM が中央編集者になっている
- 構造変化の主体が局所細胞ではない
- 生態系というより知識編集パイプラインに近い

という制約があります。

RISA はここからさらに一歩進めて、

> 中央の編集者がグラフを更新するのではなく、Concept Cell が局所ルールで動いた結果としてグラフが生まれる

ところを目標にします。

---

## 7. Knowledge Graph 系との差

一般的な Knowledge Graph や Graph AI は、RISA にとって有用な基盤ですが、中核思想は違います。

典型的な Graph AI:

- ノードとエッジを保存する
- クエリで検索する
- ルールや外部モデルで補う

RISA:

- ノードを固定データではなく Concept Cell として扱う
- 使用状況に応じて接続が代謝する
- 概念の寿命や休眠を持つ
- 構造を圧縮して新しい概念を作る

つまり、Knowledge Graph は RISA の保存形式や一時的な表現にはなり得ても、RISA の本体そのものではありません。

---

## 8. RISA の独自性

現時点で整理できる RISA の独自性は、次の 4 点です。

### 8.1 Graph を目的ではなく結果と見る

RISA では Graph は設計物ではなく、Concept Cell 群の相互作用の結果です。

### 8.2 ノードをデータではなく細胞として扱う

Concept Cell は少なくとも次を持つ存在として考える。

- 活性
- 使用回数
- エネルギー
- 近傍構造
- 寿命
- 分裂条件
- 融合条件
- 休眠条件
- 死滅条件

### 8.3 構造編集を中央命令から局所代謝へ移す

RISA の本質は `Structure Editor` ではなく、局所ルールによる構造代謝にあります。

### 8.4 AI より Artificial Life に近い

ANN や Transformer は主に計算モデルですが、RISA は

- 生まれる
- 死ぬ
- 増える
- 融合する
- 競争する
- 協調する

という意味で、人工生命的な設計へ近づいています。

---

## 9. 設計への含意

この比較から、MVP-1 以降の設計で意識したいことは次です。

### 9.1 既存 OSS を競合ではなく部品ライブラリとして見る

- ActiveGraph からはイベント駆動と関係側の振る舞い
- GraphPalace からはスティグマジー
- Active Inference からは仮説と世界モデル更新
- Knowledge Graph 系からは保存・探索・可視化

を取り込める。

### 9.2 RISA の差分は Concept Cell 層に置く

RISA 独自性はグラフストアではなく、

- 局所ルール
- 制約
- エネルギー
- 休眠
- 分裂
- 融合
- 抽象化

が働く Concept Cell 層に置くべきです。

### 9.3 MVP-1 では独自性を小さく実証する

MVP-1 で全部を作る必要はありません。  
ただし、最低限でも次のどれかは実証したい。

- 利用頻度に応じた構造変化
- 休眠と再活性化
- 類似イベント群からの概念生成
- 中央一括最適化ではない局所更新

### 9.4 長期的には Artificial Tissue として設計する

RISA の長期的な位置づけは、単なる新しい Graph AI ではなく、

> Artificial Tissue として自己組織化しながら世界モデルを育てる知能アーキテクチャ

と見るのが最も自然です。

---

## 10. 現時点での実務的な結論

研究と実装の両面で、次の姿勢が現実的です。

1. 既存 OSS の上に素朴なグラフ更新系を組む
2. その上で、Concept Cell の局所ルールを少しずつ導入する
3. 構造代謝が予測性能や継続学習で本当に効くかを測る
4. 効くなら、中央編集型から自己組織化型へ比重を移していく

つまり、最初から「完全な人工生命知能」を作ろうとするのではなく、

> 既存の Graph / Event / World Model 系を足場にしつつ、Concept Cell による自己組織化を RISA の本当の研究テーマとして育てる

という進め方が良い。
