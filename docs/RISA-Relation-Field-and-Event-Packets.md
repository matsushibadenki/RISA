# RISA 関係場とイベントパケットのメモ

## 1. 目的

この文書は、RISA を SNN と比較しながら、

- RISA では何が活性化されるのか
- RISA では何が伝播するのか
- RISA の最小計算単位は何か

を整理するための設計メモです。

現時点の主張は次です。

> RISA は「動的グラフ」よりも、「関係とイベントが共鳴・伝播する動的関係場」として捉えた方が本質に近い可能性がある

です。

---

## 2. SNN と似ている点、違う点

当初は RISA は SNN とまったく別物に見えますが、掘り下げるほど、**思想は似ていて計算対象が違う** という整理の方が近くなります。

比較すると次のようになります。

| 項目 | ANN | SNN | RISA（仮） |
| --- | --- | --- | --- |
| 活性化されるもの | ニューロン | ニューロン | 関係・イベント・構造 |
| 伝播するもの | 実数値 | スパイク | 構造活性 / イベント |
| 更新対象 | 重み | シナプス | グラフ構造 |
| 主体 | ノード | ノード | 関係 |

SNN では、

```text
Neuron A
  ->
Neuron B
  ->
Neuron C
```

のようにニューロン発火が伝播します。

RISA では、

```text
犬
  ->
哺乳類
  ->
動物
  ->
生命
```

のように、**関係の鎖そのものが活性化する** と考える方が自然です。

---

## 3. Dynamic Graph ではなく Dynamic Relation Field

これまで RISA は `Dynamic Graph` と表現してきましたが、それだけでは保存形式の説明に留まりがちです。

より本質に近い見方は、

```text
RISA = Dynamic Relation Field
```

です。

つまり、グラフは保存された構造にすぎず、実際に動いているのは **関係の流れ** です。

例えば、

```text
犬が走った
```

というイベントが入ると、

```text
犬
  ->
運動
  ->
疲労
  ->
休息
```

のような関係系列が一時的に強く活性化する。

このとき計算の主体はノードではなく、**関係ネットワーク上の動的な流れ** です。

---

## 4. 発火より共鳴

RISA を表現する言葉として、`発火` より **共鳴（Resonance）** の方が適している可能性があります。

例えば、

```text
犬
```

と

```text
猫
```

が

```text
哺乳類
```

を共有しているなら、

```text
犬
 ~~~~~
哺乳類
 ~~~~~
猫
```

のように、共有関係を介して反応が広がる。

これは単一方向のスパイク伝播というより、**関係構造の中で意味が共鳴する** イメージに近いです。

### 含意

- 活性は binary spike より連続的・構造的かもしれない
- 複数経路の交差点が重要候補になる
- 推論は共鳴が安定した部分から取り出せる可能性がある

---

## 5. 伝播するものはイベントそのものかもしれない

SNN で伝播するのが `Spike` だとすれば、RISA で伝播するのは **イベントそのもの** かもしれません。

例えば、

```text
犬が走った
```

というイベントが、

```text
運動
```

へ伝わり、

```text
疲労
```

へ伝わる。

この場合、伝播する単位は単なるオンオフ信号ではなく、

```text
Event Packet
```

と考える方が自然です。

---

## 6. Event Packet の仮説

RISA の最小計算単位はニューロンではなく、**イベントパケット** かもしれません。

イメージとしては、

```text
Event Packet
内容:
- 誰
- 何を
- いつ
- どこで
- なぜ
- 結果
```

のような構造化単位がグラフの中を移動します。

このパケットが通過した場所で、

- 活性化が起こる
- 局所推論が起こる
- 構造更新候補が生まれる

と考えることができます。

### SNN との対比

- SNN: `Spike` を飛ばす
- RISA: `Knowledge Packet` を飛ばす

という見方も可能です。

---

## 7. 問い合わせとしての伝播

RISA におけるパケットは、単なる観測イベントだけでなく、**問い合わせ** も持てる可能性があります。

例えば、

```text
犬はなぜ疲れた？
```

という問い合わせが、

```text
犬
  ->
運動
  ->
疲労
```

の関係場を探索しながら伝播し、途中で

```text
走った
```

というイベントに出会えば、それが答えになります。

この見方では、RISA の推論は単なる発火ではなく、

> 知識探索パケットの伝播

に近づきます。

---

## 8. ANN / SNN / RISA の流れの違い

### ANN

```text
入力
  ->
全部計算
  ->
出力
```

### SNN

```text
入力
  ->
必要なニューロンだけ発火
  ->
出力
```

### RISA

```text
入力
  ->
イベント生成
  ->
関係ネットワークを伝播
  ->
構造更新
  ->
推論
```

つまり RISA は、「どこが活性化するか」だけでなく、**何が伝播するか** の時点で ANN / SNN とは異なる方式です。

---

## 9. RISA の独自性は構造代謝まで含めて初めて成立する

注意すべき点があります。

RISA が単に「イベントを流すだけ」なら、新しい計算原理としては弱いです。

RISA が ANN / SNN と違う方式として成立するには、

> イベントが通過するたびに、ネットワーク自身の構造が局所的に変化し、その変化が次の推論を変える

という **構造代謝** が必要です。

つまり、

- 伝播
- 共鳴
- 局所活性化

だけでなく、

- 分裂
- 統合
- 抽象化
- 刈り込み
- 近道生成

まで含めて初めて RISA らしさが出ます。

---

## 10. 実装への含意

この整理から、今後の実装で考えるべきことは次です。

### 10.1 活性化モデル

- ノード活性ではなく関係活性をどう表すか
- 共鳴の強さをどう持つか
- 複数経路からの共鳴合流をどう扱うか

### 10.2 パケットモデル

- Event Packet の最小属性は何か
- 観測イベントと問い合わせパケットを同一形式で持てるか
- パケットが通過した経路を説明へ再利用できるか

### 10.3 構造代謝

- パケット通過時にどの条件で構造編集候補を作るか
- どの編集だけを局所評価するか
- 共鳴結果が抽象化や分裂をどう誘発するか

---

## 11. 近い将来の実装方針

MVP-1 の直後にいきなり全面導入する必要はありませんが、将来に向けて次のような段階導入が考えられます。

### Phase A

- 活性化経路を明示的に記録する
- 推論時に「どの関係が通ったか」を追跡する

### Phase B

- Event Packet に相当する問い合わせオブジェクトを導入する
- パケットが辿れる局所グラフ探索を作る

### Phase C

- 共鳴スコア
- Relation Attention
- 構造代謝トリガー

を導入し、推論と学習を結び付ける

---

## 12. 現時点の表現

現時点で RISA を一文で表すなら、次が有力です。

> ニューロンが発火する AI ではなく、イベントと関係が共鳴・伝播しながら、構造そのものを再編成していく AI

この表現は、保存構造ではなく動的過程を中心に据えられる点で有用です。

---

## 13. 結論

RISA を SNN と比較すると、両者は「局所イベントがネットワークを通じて伝播する」という意味では似ています。

しかし違いは、

- SNN はニューロン発火を扱う
- RISA は関係・イベント・構造活性を扱う

点にあります。

そして RISA の独自性は、単なる伝播ではなく、

> 伝播のたびに構造が局所的に代謝し、その結果として次の推論が変わること

にあります。

この整理は、RISA を単なる動的グラフではなく **動的関係場** として捉える方向を強く支持します。
